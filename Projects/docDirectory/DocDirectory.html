DOC DIRECTORY

The product

Doc Directory is the appication which enables you to strore the documents of any, type and can be access or get the documents whenever you needed. this application is has simple and impactful UI and UX.

Problem Statement

Accesssing document form cloud storage is not a big deal, but what when you don`t have a network or you are offline. Many cloud storage provides a good storage where we can store the documents and get them when we need them, but, there is no any application which provides us facility of offline documents. It may happen that we need a important document and we are not in the network. Many of us would have faced this problem.

The Goal

To provide the feature in application in which you can get the documents even though you are offline. This will help a lot to many people to get their documents offline,

User research summary

I conducted interviews of users and created empathy maps to understand the users mind and interact with them on what they think and what are their pain points. I have also checked that is my design is solving their problems. What users think about the design and understanding their pain points

Gathering Insights

I like to keep my documents in digital format. which can be easily accessible any time and any where.

I dont like to carry the documents all the time its better to have all the documents in one click.

In todays fast world I would prefer the online documemts. I usually travel and need the many of the documents with me.

Pain points

There is no any application which gives documents offline or in no network zone.

Unable to get the documents when network is poor.

Need to find the hotspot or good speed wifi to access the documents on cloud to get documents.

User Personas

I created two different user personas based on the target user group and all the information gather by research. I was able to create two fictional characters who represents the target user group.
